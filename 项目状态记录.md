# 项目最新状态记录

## 更新时间
2026年6月18日

## 项目概况
**引易转** (YinYiZhuan) - 中国历史学者学术引用格式转换工具

### 当前版本状态
- 最近更新：LSYJ格式英文括号→中文括号统一转换修复
- 支持12种文献类型，包括最新的古籍格式优化
- 部署在Vercel：https://yinyizhuan.vercel.app

### 核心功能
1. **多格式转换**：《历史研究》、GB/T 7714-2015、APA第7版
2. **多输入方式**：手动填写、文本解析、URL/DOI/ISBN导入、文件上传
3. **批量处理**：支持多条引用同时转换
4. **本地存储**：浏览器本地存储转换历史（最多500条）
5. **Word导出**：支持导出为Word文档

### 技术栈
- React 18 + TypeScript + Vite 5
- Tailwind CSS样式
- React Router 7路由
- opencc-js简繁转换
- docx库Word导出

### 开发注意事项
1. **中文字符处理**：TSX/JSX/HTML中必须使用真实中文字符，禁止Unicode转义
2. **Vercel部署**：必须先git push再vercel deploy，否则服务端拉取旧代码
3. **文件编码**：BibTeX/RIS解析器支持UTF-8
4. **移动端适配**：目前推荐PC端使用
5. **Layout.tsx CRLF换行**：该文件含CRLF，edit工具无法匹配，必须用write整体重写
6. **GitHub代理**：推送需要走Clash代理（端口17890）

### 构建配置
- **Vite配置**：使用React插件
- **TypeScript配置**：包含三个配置文件
  - tsconfig.json：基础配置
  - tsconfig.app.json：应用配置
  - tsconfig.node.json：Node.js配置
- **ESLint配置**：使用ESLint 9+，包含React Hooks和React Refresh插件
- **PostCSS配置**：包含Tailwind CSS和Autoprefixer

### 样式配置
项目使用Tailwind CSS，配置包括：
- **暗黑模式**：使用class策略（darkMode: 'class'）
- **自定义颜色**：
  - ink（墨色系）：深色调，用于文字和背景
  - vermilion（朱红色系）：强调色
  - parchment（羊皮纸色系）：浅色调，用于背景
- **字体**：
  - display：Noto Serif SC（衬线字体，用于标题）
  - body：Noto Sans SC（无衬线字体，用于正文）
  - mono：JetBrains Mono（等宽字体）
- **动画**：淡入、上滑、右滑动画效果

### 工具函数
项目包含多个实用工具函数：
- **generateId()** - 生成唯一ID（时间戳+随机数）
- **copyToClipboard()** - 复制文本到剪贴板
- **downloadFile()** - 下载文件到本地
- **formatTimestamp()** - 格式化时间戳
- **getCitationTypeName()** - 获取文献类型中文名称
- **getFormatName()** - 获取格式中文名称

### 主题系统
项目支持明暗主题切换：
- 使用localStorage存储主题偏好
- 默认为浅色模式（light）
- 支持系统主题自动检测
- 通过CSS类名切换主题
- 在应用启动时初始化主题

### 页面结构
项目包含5个主要页面：
1. **HomePage.tsx** - 首页，展示项目介绍和功能
2. **ConvertPage.tsx** - 核心转换页（~1019行，功能最丰富）
3. **HistoryPage.tsx** - 历史记录管理页
4. **AboutPage.tsx** - 关于页面
5. **InvitePage.tsx** - 邀请码输入页面（访问控制）

### Word文档导出
项目支持将转换记录导出为Word文档：
- 使用docx库生成格式化的引用列表
- 支持导出选中的记录
- 包含标题、格式信息、时间戳等
- 可用于学术论文参考文献整理

### 元数据自动抓取
项目支持通过DOI和ISBN自动抓取文献元数据：
- **DOI抓取**：使用Crossref API
- **ISBN抓取**：Google Books API（主要）+ Open Library API（降级fallback）
- **智能判断**：自动推断文献类型
- **数据来源标记**：记录数据来自哪个API
- **CORS支持**：可直接从浏览器前端调用，无需后端代理

### 访问控制系统
项目实现了访问控制功能：
- **试用限制**：未解锁用户可免费试用10次
- **邀请码系统**：通过邀请码解锁更多功能
- **每日限额**：解锁用户每日100次转换
- **管理员权限**：管理员无限制
- **ISBN/DOI抓取限额**：每日10次自动抓取限制
- **存储键**：使用localStorage存储各种状态

## 待推进功能清单

### 一、用户体验改进（优先级高）

| 功能 | 说明 | 复杂度 |
|------|------|--------|
| **拖拽上传** | 文件上传区域写了"点击或拖拽"但没有实现 `onDragOver`/`onDrop` | 低 |
| **批量转换** | `.bib`/`.ris` 多条目文件只能逐条选，没有"全部转换"按钮 | 中 |
| **实时预览** | 点"转换"前看不到结果，填字段时右侧实时预览会减少迭代 | 高 |
| **输入校验** | 标题/作者为空时静默输出残缺结果，应有红框提示 | 低 |
| **重置确认** | 清空表单无二次确认，误触就没了 | 低 |

### 二、格式支持扩充

| 功能 | 说明 | 复杂度 |
|------|------|--------|
| **日文引文** | 检测到 `ja` 但完全没解析逻辑，直接走英文 APA 出错 | 高 |
| **古籍/日记/经典 解析器** | 格式化器有，但 parser 永远不会自动识别这三种类型 | 中 |
| **Chicago / MLA 格式** | 目前英文只支持 APA，人文学科常用 Chicago | 高 |
| **专利/标准/报告类型** | `CitationType` 缺少 patent / standard / report | 中 |

### 三、代码质量改进

| 问题 | 说明 | 复杂度 |
|------|------|--------|
| **ConvertPage.tsx 1019行** | ManualForm、AddToGroupButton 应拆出独立文件 | 中 |
| **三处重复的 `detectLanguage`** | parser.ts / bibtex-parser.ts / ris-parser.ts 各一份 | 低 |
| **三处重复的 `authorStr`** | lsyj.ts / gbt7714.ts / apa.ts 各写各的 | 低 |
| **BibTeX 嵌套花括号解析** | 正则处理不了 `{A {Deeply} Nested {Example}}`，标准 BibTeX 常见 | 中 |
| **零测试** | parser 有大量正则逻辑，最该有测试的反而没测 | 中 |
