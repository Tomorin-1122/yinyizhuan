# 项目最新状态记录

## 更新时间
2026年6月27日

## 项目概况
**引易转** (YinYiZhuan) - 中国历史学者学术引用格式转换工具

### 当前版本状态
- 最近更新：**AGENT API 接口架构搭建完成**（5 个端点，已部署上线）
- 支持 12 种文献类型，古籍支持 7 种子类型
- 部署在 Vercel：https://yinyizhuan.vercel.app
- API 健康检查：https://yinyizhuan.vercel.app/api/health ✅

### 核心功能
1. **多格式转换**：《历史研究》、GB/T 7714-2015、APA 第7版
2. **多输入方式**：手动填写、文本解析、URL/DOI/ISBN 导入、文件上传（含拖拽）
3. **批量处理**：支持多条引用同时转换
4. **本地存储**：浏览器本地存储转换历史（最多 500 条）
5. **Word 导出**：支持导出为 Word 文档
6. **输入校验**：仅期刊强制要求标题，其他类型可不填
7. **重置确认**：清空表单前二次确认
8. **再次引证**：历史记录页支持一键复制省略出版信息的引证格式（著作 + 古籍有出版信息时）
9. **期刊页码切换**：转换结果区可切换显示/隐藏页码

### 技术栈
- React 18 + TypeScript + Vite 5
- Tailwind CSS
- React Router 7 路由
- opencc-js 简繁转换
- docx 库 Word 导出
- vitest 测试框架（58 个用例）

### 2026-06-27：AGENT API 接口架构搭建

**概述**
- 创建 `api/` 目录，实现 5 个 REST API 端点
- 所有 API 代码独立于原有前端代码，可完全回溯
- 已部署上线：`https://yinyizhuan.vercel.app/api/health` ✅

**API 端点**
| 端点 | 方法 | 功能 | 状态 |
|------|------|------|------|
| `/api/convert` | POST | 单个引用转换 | ✅ |
| `/api/batch-convert` | POST | 批量引用转换（最多50条） | ✅ |
| `/api/parse` | POST | 智能解析（只解析不格式化） | ✅ |
| `/api/formats` | GET | 获取支持的格式列表 | ✅ |
| `/api/health` | GET | 健康检查 | ✅ |

**技术细节**
- API 文件使用 **JavaScript**（非 TypeScript），避免 Vercel 编译问题
- 核心库从 `src/lib/` 复制到 `api/_lib/`，互不影响
- `vercel.json` 配置了 API 路由和 CORS 头
- 支持跨域请求（所有端点）
- 打包大小：~61 MB（含 opencc 简繁转换库）

**Vercel 部署注意事项**
- `vercel.json` 不支持 `typescript` 属性（已验证）
- Vercel 的 TypeScript 编译器要求 `.js` 扩展名，但又找不到 `.js` 文件
- **解决方案**：API 文件直接用 JavaScript，避免 TypeScript 编译问题
- 不需要 `runtime` 配置，Vercel 自动检测 Node.js

**API 文档**
- `api/README.md` - API 使用文档
- `api/IMPLEMENTATION_GUIDE.md` - 完整实现指南（含 Python/JS/Go/Java 示例）
- `api/API_USAGE_GUIDE.md` - 详细使用指南
- `api/ARCHITECTURE.md` - 架构设计图
- `api/SUMMARY.md` - 搭建总结
- `api/PUSH_REPORT.md` - 推送报告
- `api/FINAL_REPORT.md` - 最终报告

**回溯方法**

### 2026-06-22：古籍表单深度优化

**types.ts 新增字段**
- `punctuators?: Author[]` — 点校者/整理者
- `bookletVolume?: string` — 单书册数（区别于丛书册数）
- `column?: string` — 栏（影印本用）
- `category?: string` — 部类（析出文献用）
- `compileEra?: string` — 修纂年代（地方志用）
- `dynasty?: string` — 年代（古籍用，输出加方括号）

**lsyj.ts formatter 重写**
- 8 个古籍子类型独立格式化函数 + legacy 向后兼容
- `formatVolume()` — `3` → `卷3`，`435` → `卷435`，已有"卷"前缀原样
- `formatBooklet()` — `3` → `第3册`，`上册` 原样
- `formatSeriesVolume()` — `42` → `第42册`
- `dynastyPrefix()` — `[明]`、`[清]` 年代方括号前缀
- `punctuatorStr()` — 同角色合并（"王安、谢安点校"），不同角色分开
- `joinAncientParts()` — 智能拼接规则（作者/卷次后不加逗号，独立书名间加逗号）
- 刻本支持馆藏处（archiveLocation）
- 地方志支持整理者（punctuators），整理者在丛书名前输出
- 丛书名后紧跟册数（`《中国方志丛书》第1册`）

**ManualForm.tsx 表单重构**
- 古籍类型选择器移到责任者之前，常用基本典籍排最前
- 年代输入框与责任者同行（w-16），下方标注"年代选填"
- 卷次/册数/丛书册数 placeholder 改为纯数字提示（自动转换）
- 篇名提示简化为"多个层次用中圆点隔开"
- a/b 面改为下拉选择（不显示/a面/b面）
- 栏改为下拉选择（不指定/上栏/中栏/下栏）
- 版本标注：点校本/影印本/典籍改为下拉（标点本/整理本/影印本 + 自定义），刻本保留自由输入
- 点校本：多人点校者 + 责任方式下拉（点校/整理）
- 地方志：整理者（多人 + 点校/整理/不显示）
- 刻本：馆藏处输入框（可选）
- 修复版本标注自定义输入（用 useState 追踪 custom 模式）

**HistoryPage.tsx**
- 再次引证扩展到古籍类型（有出版社时显示按钮）
- 古籍再次引证输出：`[明]姚际恒：《古今伪书考》，第9页。`

**测试**
- 新增 8 个古籍子类型精确断言测试 + 向后兼容测试
- 58/58 全部通过
